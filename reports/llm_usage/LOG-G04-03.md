```text
Excellent idea. To get the maximum grade and aim for the bonus point for the best QUBO cost function, we need to design this phase very carefully. Phase 2 is the algorithmic core of the project and presents several pitfalls.

Here is an analysis of how we can structure the solution to master the requirements and optimize for the bonus.

1. Strategy for the Bonus Point (Best Optimization)
The bonus requires finding the lowest value of the cost function f(x) = -x^T Q x. Since the correlation matrix is completely dense (no correlation between feature pairs is exactly zero), the resulting QUBO is a problem on a complete graph, which is known to be particularly difficult (NP-Hard).
* The Solver: Instead of writing a custom algorithm from scratch or using standard scipy annealers, we will leverage the professional D-Wave Ocean SDK (open source). Specifically, I propose using `dwave-tabu` (Tabu Search) or `dwave-neal` (Simulated Annealing). Tabu Search is specifically known to outperform Simulated Annealing on dense graphs.
* Parameter Tuning: We will configure the solver to perform many "reads" (e.g., num_reads=1000 or more) at each iteration, keeping the solution with the lowest energy. This gives us a near-mathematical guarantee of finding a deeper minimum compared to other groups.

2. Efficient Search for the α Parameter
The specifications state that we must find an α ∈ [0, 1] that returns exactly a target number of features (K ± allowance).
* α = 0 ⇒ 0 features selected.
* α = 1 ⇒ all features selected.
* Strategy: We will not do a sequential search (e.g., trying 0.1, 0.2, 0.3...), which would waste `alpha_computations` attempts. We will implement a Binary Search (Bisection Method). We evaluate at the center (0.5): if we get too many features, we move to the lower half [0, 0.5]; if we get too few, we go to [0.5, 1]. This makes us converge on the exact value of α in very few steps (typically 6-8 iterations), leaving plenty of computation time for the solver to do a very thorough job at each step.

3. Exact Construction of the Q Matrix
The specifications (Appendix) ask to minimize:
f(x) = - [ α ∑ x_j |ρ_Vj| - (1-α) ∑_{j ≠ k} x_j x_k |ρ_jk| ]

We must map this into the standard format min(x^T Q x) accepted by D-Wave solvers:
* Diagonal elements (Q_jj): Set to - α · |ρ_Vj|. (Since x_j = x_j^2 for binary values, the influence goes on the diagonal).
* Off-diagonal elements (Q_jk): Set to + (1-α) · |ρ_jk|. (They represent the penalty if two correlated features are chosen together).
* Constructing the matrix in this precise way ensures we avoid sign inversions, which are a common mistake and would lead to maximizing the error instead of minimizing it.

4. Performance Management (1.5M Record Dataset)
The specification mentions that the evaluation dataset will have 1.5 million samples. Calculating the Spearman correlation matrix on 1.5M rows and 150 features in pure Pandas (df.corr(method='spearman')) could require a lot of RAM and several minutes.
* Strategy: We will compute the "ranks" of the columns using vectorized functions (like scipy.stats.rankdata or optimized NumPy approaches), and then calculate the Pearson correlation matrix on the ranks (which is mathematically equivalent to the Spearman correlation). This will reduce the Q matrix construction time from minutes to just a few seconds.

5. Correct Train / Test Split
The specifications ask to drop unselected features and then split the dataset (using percTest).
Reading carefully: "The division is made by considering the first M samples as the training set, and the remaining samples as the test set."
Therefore, we will avoid sklearn's classic train_test_split (which randomly shuffles data) and implement strict slicing df.iloc[:M] and df.iloc[M:] by calculating M = len(df) · (1 - percTest).
```
